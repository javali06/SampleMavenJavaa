1) Functional Requirements (FRs)


User Authentication – Register, login, logout.

CRUD Operations – Create, read, update, delete records.

Search & Filter – Find data based on criteria.

Data Processing – Validate and process user inputs.

Transactions – Complete actions like booking, payment, upload.

Notifications – Show confirmations, alerts, and messages.

Reports/Results – Generate summaries or output.

Role-Based Access – Different permissions for admin/user.



Short Non-Functional Requirements (NFRs)

Performance – Fast response, handles many users.

Security – Encryption, secure access, HTTPS.

Reliability – Stable, minimal failures.

Usability – Simple, user-friendly UI.

Availability – Accessible anytime (24/7).

Scalability – Supports growth in users/data.

Maintainability – Easy to update and debug.

Compatibility – Works on all devices/browsers.

Backup & Recovery – Prevent data loss.

Data Integrity – Accurate and consistent data.

    stage('Build') {
        echo "Building the Maven Project..."

    }

    stage('Test') {
        echo "Running Tests..."
        bat 'mvn test'
    }

    stage('Archive Results') {
        echo "Archiving Build Artifacts & Test Reports..."

        // Archive WAR/JAR files
        archiveArtifacts artifacts: 'target/*.war', fingerprint: true

        // Publish test results (Surefire Reports)
